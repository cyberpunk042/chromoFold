How to read Python vs C++

Expect parity on ported kernels, not magic. Same algorithm + same GPU means the CUDA kernel should match the Warp kernel (M1: 1233 vs ~1240 M access/s) — the native engine is not "faster because C++". Its wins come from specialization (compile-time level counts), directory redesign (two-level rank, 1.6–1.8×), entropy coding (RRR, 2.2–2.8×), the device-native API (removes the H2D/D2H tax the prototype is bound by), and fusion where the avoided intermediate is large. Absolute nanoseconds are hardware-specific; the shape — parity, ratios, scaling — is the invariant that carries to other GPUs.

Run it on your machine

Results are hardware-specific. Build on any NVIDIA GPU and the numbers slot into benchmarks/results.json as a new machine — this page extends by adding data, not rewriting.

make ARCH=sm_80 all                       # your GPU's arch
make PYTHON=~/warp-solar-system-shaders/.venv/bin/python rrr   # M4 entropy frontier
make PYTHON=... rank        # M3 two-level rank      make PYTHON=... bench    # M1 access
