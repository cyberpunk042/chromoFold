Qualification rule

A claim follows the exact artifact.

A production result must bind the runtime, model, hardware, workload fingerprint and archive digest. Changing any of them creates a new qualification target.

artifact digest + runtime + hardware + workload → PASS | FAIL | INCOMPLETE

FAIL and INCOMPLETE are retained rather than hidden.

Reproduce

Turn a benchmark into auditable evidence.

  1. Record hardware, driver, CUDA and runtime versions.
  2. Run the independent correctness reference first.
  3. Capture baseline and ChromoFold commands.
  4. Publish raw result JSON and logs.
  5. Bind results to immutable source and artifact digests.
  6. Submit through the reproduction template.
