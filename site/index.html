GPU-native compressed compute

Run more AI on the GPU you already own.

ChromoFold keeps weights, KV caches and repetitive serving data compressed in VRAM while GPU kernels access only the values they need. No full decompressed tensor is required.

The simple explanation

Like a ZIP file the GPU can use without unzipping.

Traditional

Compressed data → full decompression → large temporary tensor → computation.

ChromoFold

Compressed data → decode a needed value in registers → consume immediately.

ChromoFold trades inexpensive decoding work for lower VRAM residency and less memory traffic. It is most promising where the avoided intermediate is large, the workload is memory-bound, or only a sparse subset is consumed.

Proof path

How a recommendation becomes a claim

  1. Inspect the exact machine.
  2. Generate a profile with silent fallback disabled.
  3. Run the baseline and ChromoFold workload with identical inputs.
  4. Compare capacity, latency, throughput, VRAM and correctness.
  5. Run RC1 qualification for the exact artifact digest.
  6. Publish only PASS evidence; preserve FAIL and INCOMPLETE.
ChromoFold is open engineering: measured claims, visible caveats, reproducible evidence.
